3 contexto

3.1 enunciado

La aplicación tendrá la capacidad de programar actividades en fechas específicas, con posibilidad de recordatorios, notas desplegables y permanentes en pantalla que permitan mantener al tanto al usuario de la actividad que se programó, incluirá servicios adicionales, como el envío de e-mail, compartir información en redes sociales (twitter, facebook, myspace..etc), utilización de puntos refereciados por GPS para identificar sitios.

3.3 Caracteristicas

Eventos:

- Nuevo Evento: El sistema permitirá Ingresar actividad, programar fecha, actualizar la fecha, ubicar notas en pantalla y evitar duplicidad de información.

- GPS: El sistema permitirá buscar el sitio de la actividad.

- WEB: El sistema permitirá enviar correo electrónicoy publicar en redes sociales

Recordatorio:

Recordar Eventos: El sistema permitirá programar la intensidad, consultar la actividad y alertar sobre actividades proximas.

Editar Eventos:El sistema permitirá la edición de actividades

Tipo de Evento:

Clasificar Eventos: El sistema permitirá elegir entre varias clases de eventos

Versión 1.0 ( 28/10/2012 )
Autores
  • Diego Correa
Fuentes
  • Andres Ramirez
Dependencias Ninguno
Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el usuario decide crear la actividad
Precondición El usuario ya debió ingresar a la aplicación.
Secuencia normal Paso Acción
1 El sistema presenta el formulario para ingresar la actividad
2 El actor Usuario (ACT-0001) ingresa el nombre de la actividad que requiere
3 El actor Usuario (ACT-0001) selecciona la intensidad de la actividad (Diaria - Semanal - Mensual)
4 El actor Usuario (ACT-0001) programa la fecha de la actividad.
5 El actor Usuario (ACT-0001) clasifica la actividad de acuerdo a la importancia que requiera.
6 El actor Usuario (ACT-0001) ingresa los datos del sitio tales como país, departamento, ciudad y dirección.
7 El actor Usuario (ACT-0001) procede a guardar la información
8 Se realiza el caso de uso Enviar correo electrónico (UC-0005)
Postcondición la actividad es creada
Excepciones Paso Acción
7 Si el sistema encuentra una actividad para la misma fecha y hora seleccionada, el sistema presenta un mensaje informando que ya existe una actividad para ese mismo rango, a continuación este caso de uso queda sin efecto
Rendimiento Paso Tiempo máximo
